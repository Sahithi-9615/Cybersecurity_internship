How passwords are stored

1
Input received

Plain-text password arrives server-side

2
Argon2id hashing

Memory: 64MB · Iterations: 3 · Parallelism: 4

3
Salt generated

16-byte cryptographic salt per user

4
Stored in DB

Only hash + salt — never plaintext

Server-side logic (Node/Express)

  type: argon2.argon2id,
  memoryCost: 2 ** 16,
  timeCost: 3,
  parallelism: 4
});

Reset flow security

Max 5 requests/hour per email

const raw = crypto.randomBytes(64).toString('hex');
const hash = crypto.createHash('sha256')
  .update(raw).digest('hex');
await db.tokens.create({
  hash,
  userId,
  expiresAt: Date.now() + 15 * 60000
});

Middleware enforcement

const authorize = (permission) => (req, res, next) => {
  const { role } = req.user; // from JWT
  const perms = ROLE_PERMISSIONS[role];
  if (!perms?.includes(permission)) {
    return res.status(403).json({
      error: 'Forbidden'
    });
  }
  next();
};

// Usage on a route:
app.delete('/users/:id',
  authenticate, authorize('users:delete'),
  deleteUserHandler);
